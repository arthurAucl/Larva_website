Program with Larva


The LARVA architecture involves the following five components:

(i) a system to be monitored;
(ii) a set of specifications written in some formal notation;
(iii) a stream of events extracted from the system in (i);
(iv)a monitoring system which receives the events and verifies them according to the specification in (ii);
(v) possibly a feedback loop.

We will try to learn the main elements to program the (ii).

To define the different states of a state machine, we need the STATES construction. Here we build the "BAD" state that defines and calls toomany{} and baddelete{}.
We also have the creation of the "NORMAL" state as well as the "STARTING" state. The latter uses the start function which sends a message to the terminal.
Unlike the start function, the other functions are just set but do nothing.



With the construction "TRANSITIONS", it is possible to determine the conditions to pass from one state to another as well as the functions to be called.

Another defining element of Larva is the foreach construct which allows top level universal quantification in specifications in a straightforward manner.
